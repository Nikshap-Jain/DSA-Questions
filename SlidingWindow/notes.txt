class Solution {
    public int slidingWindowTemplate(int[] nums, int k) {
        int l = 0;
        int sum = 0;
        int ans = 0;

        // 1. Always expand 'r' (right pointer) one by one
        for (int r = 0; r < nums.length; r++) {
            // Add the new element to the window
            sum += nums[r];

            // 2. Shrink 'l' (left pointer) IF sum becomes too big
            // (Note: This only works if numbers are positive!)
            while (sum > k && l <= r) {
                sum -= nums[l];
                l++;
            }

            // 3. Check if we found the target
            if (sum == k) {
                ans++;
            }
        }
        return ans;
    }
}

Key Takeaway
The question to always ask yourself is:

"If I add one element to the right or remove one element from the left, can I update my window's condition in O(1) time?"

If the answer is no (either because of negative numbers or a complex check like a palindrome), it's probably a trap.

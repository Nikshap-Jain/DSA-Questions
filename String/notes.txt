# Java String & Character Methods - Quick Reference



String Methods for leetcode- 

The Java String class is a beast, but for LeetCode, you're really just using the same few tools from the toolbox. Focus on these, and you'll be set for 99% of string problems.
Here are the heavy hitters, grouped by why you'd use them.

1. Basic Inspection (The "Read-Only" Ops)

These are your non-negotiable basics for checking what's inside.
* length(): Returns the number of characters. You'll use this in basically every for loop.
* charAt(int index): Gets the character at a specific position. The other half of your for loop.
* isEmpty(): Quick check if length() == 0. Good for edge cases.

2. Slicing, Dicing, and Converting

When you need to break the string apart or turn it into something else.
* substring(int beginIndex, int endIndex): Extracts a new string. endIndex is exclusive. This is your go-to for all substring and sliding window problems.
* toCharArray(): (Very Important) Converts the string to a char[]. This is your escape hatch when you need to modify or sort the characters (e.g., Anagram problems). Remember, String itself is immutable.
* split(String regex): Breaks a string into an array of strings based on a delimiter (like " ", ",", or "\\."). Perfect for parsing input.

3. Searching and Comparing

For finding needles in a haystack or checking for equality.
* equals(Object anObject): Checks if two strings have the exact same characters. Don't get caught by the == vs .equals() trap. Always use .equals() for value comparison.
* indexOf(String str) / indexOf(char ch): Finds the first occurrence of a character or substring. Returns -1 if not found. Great for "find first..." problems.
* contains(CharSequence s): A quick true/false check if a substring exists.
* startsWith(String prefix) / endsWith(String suffix): Exactly what they sound like. Useful for validation.

4. Cleaning and Formatting

Primarily used for case-insensitive problems or handling messy input.
* toLowerCase() / toUpperCase(): Returns a new string in the specified case. Essential for problems like "Valid Palindrome" where case doesn't matter.
* trim(): Removes whitespace from the beginning and end.


‚ö° The Most Important "String" Tool (That Isn't in String)

This is the one that will save you from "Time Limit Exceeded" (TLE).
StringBuilder
Strings are immutable. This means every time you do s = s + "a"; inside a loop, you are creating a brand new string and copying the old one. This is slow ($O(n^2)$) and a classic LeetCode trap.
When you need to build a new string piece-by-piece, always use StringBuilder.
Java



// The SLOW way (will get TLE on many problems)
String result = "";
for (char c : someArray) {
    result += c; // Creates a new string every single time. Bad.
}

// The FAST way (correct)
StringBuilder sb = new StringBuilder();
for (char c : someArray) {
    sb.append(c); // Modifies the StringBuilder in-place. Good.
}
String result = sb.toString(); // Convert back to string once at the end.
StringBuilder also has other handy methods like:
* sb.append(String str)
* sb.reverse() (A lifesaver for palindrome problems)
* sb.deleteCharAt(int index)

üéÅ The Bonus Helpers

These aren't String methods, but you'll use them constantly with strings.
* Integer.parseInt(String s): Converts a numeric string (like "123") to an int.
* String.valueOf(int n): Converts an int (or char, double, etc.) to a String.
* Arrays.sort(char[] a): Use this after toCharArray() to check for anagrams.

When comparing strings in Java, always, always, always use .equals(). 
* Use == for primitive types (like int, boolean, char).(it check if object address is same not the content)
* Use .equals() for objects (like String, Stack, Integer).(it check the content this is what we want)




## Character Class Methods

Character.isUpperCase(ch)  // Check if char is uppercase
Character.isLowerCase(ch)  // Check if char is lowercase
Character.isDigit(ch)      // Check if char is digit
Character.isLetter(ch)     // Check if char is letter


## String Case Methods

str.toUpperCase()          // Convert to uppercase
str.toLowerCase()          // Convert to lowercase
str.equals(str.toUpperCase())  // Check if string is all uppercase
str.equals(str.toLowerCase())  // Check if string is all lowercase


## Important Notes
- **String class has NO** `isUpperCase()` or `isLowerCase()` methods
- Use `Character` class for single character operations
- For strings, use `equals()` with `toUpperCase()/toLowerCase()` for case checking
- String concatenation in loops is inefficient - consider `StringBuilder`
